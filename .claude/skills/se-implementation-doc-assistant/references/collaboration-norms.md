# GitLab 协作与分工规范记录示例

本文件作为 Claude 生成“协作过程”和“分工权重”的参考基准。

## 1. 分支策略说明
- `main`: 仅保存可上线的版本，受保护。
- `develop`: 核心开发分支。
- `feature/*`: 功能分支，命名规则：`feature/F[需求号]-[简述]`。
- `fix/*`: 缺陷分支，命名规则：`fix/B[缺陷号]-[简述]`。

## 2. 角色分工 (参考课程夏季模板)
| 角色 | 核心职责 | 考核权重 |
| :--- | :--- | :--- |
| **组长/项目经理** | 需求规划、任务分发、合并 PR、风险管理 | 1.1 - 1.2 |
| **核心开发** | 关键框架搭建、难点模块攻克、数据层设计 | 1.0 - 1.1 |
| **测试负责人** | 测试用例编写、集成测试环境维护、UI 还原度检查 | 0.9 - 1.0 |
| **文档与协调** | 文档规范维护、进度追踪记录、PPT 汇报整理 | 0.9 |

## 3. 合并请求 (Merge Request) 规范
- 必须包含：解决了哪个需求、涉及哪些核心改动、自测截图/Log。
- 至少一名非开发者进行代码评审 (Review)。

## 4. 缺陷管理周期
- 提交：Bug 状态、复现步骤、严重程度。
- 修复：修复分支、解决耗时。
- 验收：测试人员验证通过后关闭。
