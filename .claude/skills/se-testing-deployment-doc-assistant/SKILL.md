# SE Testing Deployment Doc Assistant

将测试与部署活动形成“可复现、可验证、可交付”的课程文档。

## Overview

将测试与部署活动形成“可复现、可验证、可交付”的课程文档。

### 适用模板

春季优先：
- `assets/测试报告.md`
- `assets/用户手册模版.md`

夏季优先：
- `assets/用户使用说明书.md`
- `assets/项目总结.md`（测试与部署复盘章节）

### 输入

- 测试计划与测试用例
- 自动化测试输出/手工测试记录
- 缺陷清单（状态、严重级别、修复版本）
- 部署环境、部署脚本、运行截图

### 输出要求

- 测试范围、方法、结果统计
- 关键缺陷闭环情况
- 部署步骤、回滚策略、运行验证
- 用户操作说明与常见问题
- 已知限制与后续优化

## Workflow

### 1. 测试设计与执行对齐

- **用例追踪**: 关联需求编号，覆盖功能/性能/兼容/安全等多维度。
- **结果统计**: 记录实际执行结果、偏差情况及原因。

### 2. 缺陷闭环分析

- **高优缺陷记录**: 包含复现条件、根因分析、修复方式及回归结果。

### 3. 部署文档化

- **环境要求**: 部署命令、参数、配置项说明。
- **健康检查**: 运行验证步骤及回滚计划。

### 4. 用户手册整理

- **操作指南**: 角色权限、业务流程操作、错误提示与处理建议。
- **常见问题**: FAQ 汇总。

### 质量检查与输出风格

- **事实优先**: 避免“基本完成”等模糊表达，指标必须量化（通过率、缺陷密度、部署耗时）。
- **可复现性**: 部署步骤必须确保他人可复现。
- **风险管理**: 对未解决问题给出风险等级与处置计划。
