### 1.2 系统概述

"饱了么"是由饕餮科技有限公司开发的一款**专注于校园场景的外卖下单平台**，秉持"让校园生活更便捷"的理念，面向高校师生、校内餐饮商家及校园配送骑手三类核心角色提供服务。

系统覆盖以下核心业务链路：
- **学生用户**：浏览/搜索店铺与商品 → 下单购买 → 实时追踪配送 → 评价
- **商家**：创建/管理店铺与商品 → 接收订单 → 更新备餐状态 → 数据分析
- **骑手**：抢单/接单 → 导航取餐 → 配送完成
- **管理员**：审核商家资质与菜品 → 管理订单与评价 → 平台内容治理

参考系统：美团外卖（https://waimai.meituan.com/）、饿了么（https://www.ele.me/）、校园食堂系统（https://www.xuexiaoshipin.com/）

**投资方 / 需方**：饕餮科技有限公司
**用户**：高校在校学生（主要）、教职工（次要）、商家、骑手、平台管理员
**开发方**：饕餮科技有限公司研发团队
**计划运行现场**：校园内网及公网混合部署，支持移动端（iOS/Android）和 Web 端

### 3.2 需求概述

#### 3.2.1 目标

**现有问题**：校园内缺乏专属外卖平台，学生点餐依赖通用平台，存在配送范围不精准、校内地址支持差、商家与学生互动弱等问题。

**本系统解决的问题**：
- 为学生提供校园场景专属的便捷点餐入口
- 为商家提供高效接单与销售数据分析工具
- 为骑手提供智能派单与校内导航支持
- 为平台管理员提供内容审核与治理工具

**主要功能模块**：

```
┌─────────────────────────────────────────────────────────┐
│                     饱了么平台                           │
├──────────┬──────────┬──────────┬────────────────────────┤
│ 用户点餐  │ 商家接单  │ 骑手配送  │    审核与管理           │
│          │          │          │                         │
│ ・主页推荐 │ ・店铺管理 │ ・智能派单 │ ・商家资质审核           │
│ ・搜索筛选 │ ・商品管理 │ ・抢单模式 │ ・菜品内容审核           │
│ ・购物车  │ ・订单处理 │ ・导航配送 │ ・订单管理              │
│ ・支付下单 │ ・数据看板 │ ・状态同步 │ ・评价管理              │
│ ・个人中心 │ ・促销设置 │          │ ・店铺/商品下架          │
│ ・优惠券  │          │          │                         │
│ ・拼单分享 │          │          │                         │
└──────────┴──────────┴──────────┴────────────────────────┘
```

**外部接口关系图**：

```
用户(APP/Web) ←→ 饱了么后端服务 ←→ 地图/导航 API
                      ↕                  ↕
                  数据库集群          支付网关
                      ↕
                  消息推送服务
```

#### 3.2.2 运行环境

| 环境项 | 要求 |
|--------|------|
| **客户端** | iOS 13+、Android 8+、主流浏览器（Chrome 90+、Safari 14+） |
| **服务端操作系统** | Linux（Ubuntu 20.04 LTS 或以上） |
| **数据库** | 关系型数据库（MySQL 8.0+）+ 缓存（Redis 6.0+） |
| **网络** | 校园内网 + 公网，支持 HTTPS |
| **部署方式** | 容器化部署（Docker/Kubernetes） |

#### 3.2.3 用户的特点

| 用户类型 | 特点 | 技术熟悉度 |
|----------|------|-----------|
| **学生用户** | 18-25岁大学生，熟悉移动互联网，追求效率与社交体验 | 高 |
| **商家** | 校内餐饮从业者，年龄跨度大，使用平板/手机接单为主 | 中 |
| **骑手** | 校内兼职学生或专职配送人员，需快速掌握操作 | 中 |
| **管理员** | 平台运营人员，具备一定系统操作能力 | 高 |

#### 3.2.4 关键点

1. **实时性**：订单状态、配送位置需实时同步（WebSocket 或长轮询）
2. **智能派单算法**：基于骑手位置、订单分布进行最优分配
3. **地图集成**：骑手导航路径规划与用户实时定位追踪
4. **高并发处理**：就餐高峰（11:00-13:00、17:00-19:00）并发订单量大
5. **数据安全**：用户支付信息、个人信息需加密存储与传输

| **技术约束** | 团队技术栈以 Java/Python 后端 + React/Vue 前端为主 |
| **地域约束** | 初期仅服务于指定高校校园范围内 |

### 3.4 CSCI能力需求

#### 3.4.1 用户注册与登录

**FR-001**：用户注册
- **触发条件**：用户首次使用平台，点击"注册"按钮
- **处理规则**：支持手机号+验证码注册；手机号格式校验（11位国内号码）；验证码60秒有效期，每日同号最多发送5次
- **成功输出**：账号创建成功，自动登录，跳转主页
- **异常处理**：手机号已注册提示"该手机号已注册，请直接登录"；验证码错误提示"验证码错误，请重新输入"
- **验收标准**：注册成功后用户可正常登录；连续5次验证码错误后锁定10分钟
- **状态**：[已确认]

**FR-002**：用户登录
- **触发条件**：用户打开 App/Web，输入账号密码或手机验证码
- **处理规则**：支持手机号+密码、手机号+验证码两种方式；密码错误5次后账号锁定30分钟
- **成功输出**：登录成功，返回 Token，跳转主页
- **异常处理**：账号不存在提示"该账号不存在"；密码错误提示剩余尝试次数
- **验收标准**：登录响应时间 ≤ 2秒；Token 有效期 7 天
- **状态**：[已确认]

#### 3.4.2 主页与推荐

**FR-010**：主页展示
- **触发条件**：用户登录后进入主页
- **处理规则**：展示推荐店铺列表（≥5家）和推荐商品列表（≥10个）；推荐逻辑基于用户历史浏览/购买记录和全平台热销数据
- **成功输出**：主页加载完成，展示推荐内容
- **异常处理**：新用户无历史数据时展示平台热销内容
- **验收标准**：主页首屏加载时间 ≤ 3秒（4G 网络环境）
- **状态**：[已确认]

**FR-011**：搜索功能
- **触发条件**：用户在搜索框输入关键词并确认
- **处理规则**：支持按商品名称、店铺名称模糊搜索；搜索结果按相关度排序；支持搜索历史记录（最多保留20条）
- **成功输出**：返回匹配的店铺和商品列表
- **异常处理**：无结果时提示"未找到相关内容，换个关键词试试"
- **验收标准**：搜索响应时间 ≤ 1.5秒
- **状态**：[已确认]

#### 3.4.3 店铺浏览与筛选

**FR-020**：店铺列表筛选
- **触发条件**：用户进入店铺浏览页，使用筛选条件
- **处理规则**：支持以下筛选维度（可多选）：
  - 店铺类型（快餐、饮品、正餐、小吃等）
  - 店铺位置（校区/楼栋）
  - 店铺评分（4.5分以上、4.0分以上等）
  - 价格区间（¥0-15、¥15-30、¥30+）
  - 配送时间（30分钟内、45分钟内）
- **成功输出**：按筛选条件过滤后的店铺列表
- **异常处理**：无符合条件店铺时提示"暂无符合条件的店铺"
- **验收标准**：筛选结果响应时间 ≤ 1秒
- **状态**：[已确认]

**FR-021**：店铺详情页
- **触发条件**：用户点击某一店铺
- **处理规则**：展示店铺基本信息（名称、评分、配送时间、配送费、起送价）、热销商品（前5个）、完整商品列表（含分类）、商品价格
- **成功输出**：店铺详情页完整展示
- **异常处理**：店铺被下架时展示"该店铺暂停营业"
- **验收标准**：详情页加载时间 ≤ 2秒
- **状态**：[已确认]

**FR-022**：商品详情页
- **触发条件**：用户点击某一商品
- **处理规则**：展示商品名称、图片、描述、价格、用户评价列表（按时间倒序，每页10条）
- **成功输出**：商品详情页完整展示
- **异常处理**：商品已下架时展示"该商品已下架"并推荐相似商品
- **验收标准**：详情页加载时间 ≤ 2秒
- **状态**：[已确认]

#### 3.4.4 购物车与下单

**FR-030**：购物车管理
- **触发条件**：用户在商品详情页点击"加入购物车"
- **处理规则**：购物车按店铺分组；不同店铺商品不可混合结算；支持修改数量（最小1，最大99）、删除单品、清空购物车
- **成功输出**：购物车更新成功，展示最新商品列表和合计金额
- **异常处理**：商品已下架时提示"该商品已下架，请移除后继续"
- **验收标准**：购物车数据跨设备同步，操作响应时间 ≤ 1秒
- **状态**：[已确认]

**FR-031**：下单与支付
- **触发条件**：用户确认购物车内容，点击"去结算"
- **处理规则**：
  1. 选择配送地址（校园内地址，支持收藏常用地址）
  2. 选择支付方式（在线支付，支持微信支付/支付宝）
  3. 可选择使用优惠券（自动筛选可用券）
  4. 确认订单总额后提交
- **成功输出**：订单创建成功，跳转支付页面；支付完成后展示订单追踪页
- **异常处理**：支付超时（15分钟内未支付）自动取消订单；库存不足提示
- **验收标准**：下单到支付完成全流程 ≤ 60秒；订单创建成功率 ≥ 99.9%
- **状态**：[已确认]

**FR-032**：订单追踪
- **触发条件**：用户支付成功后，或在订单列表点击进行中的订单
- **处理规则**：实时展示订单状态（待接单→商家备餐中→骑手已取餐→配送中→已完成）；展示骑手实时位置（地图）；支持催单（每笔订单限催1次）
- **成功输出**：订单追踪页实时刷新，位置更新间隔 ≤ 10秒
- **异常处理**：定位失败时提示"当前无法获取骑手位置"
- **验收标准**：状态更新延迟 ≤ 10秒；地图渲染时间 ≤ 3秒
- **状态**：[已确认]

#### 3.4.6 商家店铺管理

**FR-050**：商家注册与资质提交
- **触发条件**：商家申请入驻平台

### 3.18 算法说明

**智能派单算法（概况）**：
- 输入：待分配订单集合、在线骑手集合（含位置、当前承接订单数）
- 评分公式：`综合得分 = α × (1/距离) + β × (1/当前承接数) + γ × 骑手历史评分`
- 权重 α、β、γ 为可配置参数，初始值待运营调优 [待确认]
- 输出：为每个订单推荐得分最高的骑手

**推荐算法（概况）**：
- 新用户：基于全平台热销商品和新用户默认偏好
- 老用户：基于用户历史浏览、购买行为的协同过滤（简化版）

| 学生用户 | 高峰期 ≤ 3,000 | 高（就餐时段密集） |
| 商家 | ≤ 200 | 中（就餐时段持续监控） |
| 骑手 | ≤ 500 | 高（持续使用导航和状态更新） |
| 管理员 | ≤ 10 | 低（按需操作） |

系统需提供帮助中心（FAQ）和新手引导，降低用户学习成本。

---

### 3.20 有关培训需求

- 面向学生用户：App 内置新手引导（首次使用时展示，可跳过）
- 面向商家：提供商家操作手册（PDF）和视频教程
- 面向骑手：骑手端内置操作引导
- 面向管理员：提供后台管理操作培训文档

---

### 3.21 有关后勤需求

- 系统维护窗口建议选择凌晨 2:00-5:00（非就餐时段），维护前提前公告
- 软件更新采用滚动发布（Rolling Update），尽量不影响在线用户
- 数据库每日全量备份 + 实时 binlog 备份，备份保留30天

---

### 3.22 其他需求

**FR-090**：评价系统
- 用户在订单完成后可对商品和配送进行评价（评分1-5星 + 文字 + 图片）
- 评价后可见，不可修改（避免恶意刷评）
- 商家可回复评价
- **验收标准**：评价提交后5秒内可见；图片大小限制 ≤ 5MB/张，最多3张

**FR-091**：消息通知
- 订单状态变更（接单、取餐、送达）需推送 App 消息通知给用户
- 新订单推送给商家（声音+弹窗+角标）
- 骑手收到派单/抢单成功通知

---

### 3.23 包装需求

本系统为移动 App + Web 应用，无物理介质交付需求。交付物包括：
- 移动端 App 安装包（iOS: .ipa，Android: .apk/.aab）
- Web 前端静态资源包
- 后端服务 Docker 镜像
- 部署文档、用户手册、接口文档

## 7 注解

### 术语与定义

| 术语 | 定义 |
|------|------|
| CSCI | 计算机软件配置项（Computer Software Configuration Item） |
| SRS | 软件需求规格说明书（Software Requirements Specification） |
| MVP | 最小可行产品（Minimum Viable Product） |
| RBAC | 基于角色的访问控制（Role-Based Access Control） |
| JWT | JSON Web Token，用于认证的令牌机制 |
| P95 | 第95百分位响应时间，即95%请求的响应时间 |
| TPS | 每秒事务数（Transactions Per Second） |
| WebSocket | 一种全双工通信协议，用于实时数据推送 |
| 智能派单 | 系统根据算法自动将订单分配给最合适骑手的机制 |
| 抢单 | 骑手主动选择并接受待配送订单的模式 |
| 拼单 | 多个用户合并为一个订单共同下单的功能 |
| 优惠券 | 平台发放的可抵扣订单金额的电子凭证 |
| 校园美食榜单 | 系统周期性生成的校园内热门餐饮排行榜 |
