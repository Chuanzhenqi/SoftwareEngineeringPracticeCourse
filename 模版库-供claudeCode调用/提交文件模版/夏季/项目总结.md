 ```markdown
## 一、项目成果概述

### （1）项目定位
**格式**: 1-2段话描述项目核心目标与价值主张

### （2）功能清单
**格式**: 三级结构（功能模块 → 功能点 → 特性描述）

 ```
1. {模块一名称}
   • {核心功能点}
   • {核心功能点}
   
2. {模块二名称}
   • {核心功能点}
```

### （3）技术架构
**格式**: 分点列出技术栈，标注选型理由

| 层级 | 技术选型 | 用途说明 |
|-----|---------|---------|
| 前端 | {框架/库} | {用途} |
| 后端 | {框架/库} | {用途} |
| 数据库 | {类型} | {用途} |
| 部署 | {平台/工具} | {用途} |

### （4）项目成果与迭代方向
**格式**: 
- 成果： bullet points 列举可交付物
- 未来： bullet points 列举改进方向

## 二、项目反思

### （1）团队协作与管理
**格式**: 分工具/方法维度，每维度包含：
- ✅ 可复用的经验
- ⚠️ 待改进的问题
- 💡 改进建议/后续行动

### （2）技术选型
**格式**: 分框架/工具维度，每维度包含：
- 选型背景
- 遇到的问题（附具体案例）
- 经验教训

### （3）工程实践
**格式**: 按阶段（设计/开发/测试/部署）或主题（架构/代码质量/文档）反思

---

## 三、总结与展望

### 核心收获
**格式**: 3-5条 bullet points，涵盖技术能力、协作经验、工程思维

### 关键教训
**格式**: 编号列表，每条包含问题描述 + 根因分析 + 预防措施

### 后续行动
**格式**: 可量化的改进目标，如：
- 短期（下次项目）：{具体行动}
- 长期（能力沉淀）：{具体行动}
```

---

## 格式规范说明

| 元素      | 格式                  | 用途                         |
| --------- | --------------------- | ---------------------------- |
| 主标题    | `#`                   | 项目名称                     |
| 章节标题  | `##` / `###`          | 一、二、三 / （1）（2）（3） |
| 元信息    | `> **xxx**:`          | 项目周期、规模等关键信息     |
| 经验/问题 | `**加粗标题**` + 正文 | 突出关键工具或方法           |
| 引用/金句 | `>`                   | 名人观点、团队共识           |
| 步骤流程  | `1. 2. 3.`            | 改进建议、行动计划           |
| 对比表格  | `\| 列1 \| 列2 \|`    | 技术栈对比、前后对比         |

---

## 可选扩展章节

按需添加：

- **数据指标** — 性能数据、用户数据、质量数据
- **风险记录** — 遇到的风险、应对措施、实际结果
- **知识沉淀** — 技术文档、工具脚本、最佳实践
- **致谢** — 特别贡献者、外部支持
